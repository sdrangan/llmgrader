# Developer Guide

This section is for contributors and maintainers working on the `llmgrader` codebase itself.
A developer will not typically create a separate course material repository, but rather work in the `llmgrader` repository itself.

Topics here focus on repository-level development workflows, local tooling, editor integration, and contributor-facing setup rather than day-to-day course
authoring.
